# Actero Voice Agent (ElevenLabs Conversational AI)

Endpoints under `/api/voice/*` wire a client's Shopify-focused brain to an
ElevenLabs Conversational AI agent. ElevenLabs handles the whole telephony
stack (number rental, Twilio, streaming STT/TTS). We only expose a tiny
OpenAI-compatible Custom LLM so every spoken turn is generated by our
existing `runBrain()`.

## Endpoints

| Method | Path                          | Auth                         | Purpose                                           |
| ------ | ----------------------------- | ---------------------------- | ------------------------------------------------- |
| POST   | `/api/voice/custom-llm`       | `X-Voice-Auth-Token` header  | OpenAI-compatible LLM called by ElevenLabs        |
| POST   | `/api/voice/setup-agent`      | Supabase JWT (Bearer)        | Provision the ElevenLabs agent for a client      |
| POST   | `/api/voice/update-agent`     | Supabase JWT (Bearer)        | Update prompt / voice / greeting                  |
| POST   | `/api/voice/delete-agent`     | Supabase JWT (Bearer)        | Remove agent + disable voice feature              |
| GET    | `/api/voice/list-numbers`     | Supabase JWT (Bearer)        | List ElevenLabs-managed phone numbers             |
| POST   | `/api/voice/attach-number`    | Supabase JWT (Bearer)        | Attach a phone number to the client agent         |
| POST   | `/api/voice/test-call`        | Supabase JWT (Bearer)        | WebRTC signed URL to test the agent in-browser    |

All JWT-protected endpoints verify that the authenticated user has access to
the requested `client_id` via `client_users`.

## Required environment variables

| Variable                       | Required | Description                                                                                                      |
| ------------------------------ | -------- | ---------------------------------------------------------------------------------------------------------------- |
| `ELEVENLABS_API_KEY`           | yes      | Workspace API key used to manage agents, voices, numbers and signed URLs                                         |
| `ELEVENLABS_DEFAULT_VOICE_ID`  | yes      | Default French voice used when a client has not picked their own                                                 |
| `VOICE_LLM_SECRET`             | yes      | Shared secret sent by ElevenLabs as `X-Voice-Auth-Token` on every Custom LLM call. Generate a long random string |
| `PUBLIC_API_URL`               | yes      | Public URL of the app (e.g. `https://actero.fr`) — used to build the Custom LLM URL passed to ElevenLabs         |
| `SUPABASE_URL` / `SUPABASE_SERVICE_ROLE_KEY` | yes | Used by the service-role Supabase client                                                                         |
| `ELEVENLABS_WEBHOOK_SECRET`    | optional | Used by `/api/engine/webhooks/elevenlabs-postcall` to verify the post-call HMAC                                  |

## Custom LLM contract

ElevenLabs calls `POST {PUBLIC_API_URL}/api/voice/custom-llm?client_id=<uuid>`
with an OpenAI `chat/completions` payload
(`{ messages: [...], model, stream }`) and header
`X-Voice-Auth-Token: <VOICE_LLM_SECRET>`.

The endpoint:

1. Authenticates via the shared secret.
2. Resolves `client_id` from the query string.
3. Loads (or falls back to) a playbook and calls `runBrain()`.
4. Sanitizes the answer for TTS (no markdown, no URLs, max ~500 chars).
5. Replies in the OpenAI format — JSON completion or SSE stream if
   `stream: true`.

Reference docs:
<https://elevenlabs.io/docs/conversational-ai/customization/llm/custom-llm>

## Storage

- `client_settings` now carries: `voice_agent_enabled`,
  `elevenlabs_agent_id`, `elevenlabs_voice_id`, `voice_phone_number`,
  `voice_phone_number_id`, `voice_greeting`, `voice_business_hours`.
- `voice_calls` has been enriched with `elevenlabs_call_id`, `caller_phone`,
  `sentiment_score`, `classification`, `cost_usd`, `audio_url`, `ended_at`
  (existing columns are preserved for backwards compatibility).
- Every finished call also lands in `automation_events` as
  `voice_call_completed` so dashboard KPIs pick it up automatically.

title: "Connecter Shopify à Actero en OAuth 1-clic"

Shopify est l'intégration phare d'Actero. Une fois connectée, ton agent répond aux questions WISMO ("où est ma commande ?"), retourne, échange, suivi de panier, recommande des produits et personnalise chaque réponse avec le profil client — sans toucher à ta caisse.

## Prérequis

- Une boutique Shopify active (plan Basic, Shopify, Advanced ou Plus — peu importe)
- Le rôle **Owner** ou **Staff avec permission "Apps"** dans ton compte Shopify
- Un compte Actero avec au moins le plan **Free** (50 tickets/mois) — plan **Starter** ou supérieur recommandé pour les webhooks paniers abandonnés

## Connexion (OAuth 1-clic)

<Steps>
  <Step title="Ouvre Intégrations dans Actero">
    Depuis ton dashboard, va dans **Intégrations → Plateforme e-commerce → Shopify** et clique **Connecter**.
  </Step>
  <Step title="Renseigne ton domaine Shopify">
    Saisis l'URL exacte de ta boutique au format `ma-boutique.myshopify.com` (pas ton domaine personnalisé). Tu le retrouves dans Shopify Admin → **Settings → Domains**.
  </Step>
  <Step title="Autorise Actero sur Shopify">
    Tu es redirigé vers Shopify pour valider les scopes. Approuve la demande. Aucune carte bancaire requise — Actero est gratuit côté Shopify App Store.
  </Step>
  <Step title="Retour automatique">
    Tu reviens dans Actero avec un badge **Connecté** et le nom de ton domaine affiché. Les 30 dernières commandes sont indexées en moins de 20 secondes.
  </Step>
</Steps>

<Tip>
Si tu n'as pas encore configuré ton agent, regarde le [quickstart](/essentials/quickstart) — c'est la suite logique de cette connexion.
</Tip>

## Ouvrir Actero depuis Shopify Admin

Une fois l'app installée, la tuile **Actero** apparaît dans Shopify Admin → **Apps**. Cliquer sur la tuile t'amène directement au dashboard Actero, déjà scopé sur ta boutique — pas besoin de retaper ton email ni de choisir un tenant.

Sous le capot, Shopify redirige vers `https://actero.fr/api/shopify/app` avec une signature HMAC. Actero :

1. Vérifie la signature HMAC pour s'assurer que la redirection vient bien de Shopify.
2. Cherche le `shop_domain` dans la table des connexions.
3. Si la boutique est **déjà connectée**, te renvoie sur `/dashboard?shop=ma-boutique.myshopify.com`.
4. Si la boutique **n'est pas encore connectée** (cas rare — installation interrompue), te renvoie sur le flow OAuth `/api/shopify/install`.

<Note>
Tu n'as rien à configurer côté Actero pour activer ce raccourci — il est câblé automatiquement à la première installation via le Shopify App Store.
</Note>

## Scopes demandés

Shopify sépare désormais les scopes **requis** (impossibles à refuser sans annuler l'installation) et les scopes **optionnels** (que tu peux décliner — l'agent fonctionne quand même, en mode dégradé sur les features concernées).

### Requis

| Scope | À quoi ça sert |
|---|---|
| `read_orders`, `read_fulfillments` | Réponses WISMO, statut tracking, ETA livraison |
| `read_customers` | Identifier le client qui pose la question, personnaliser le ton |
| `read_products`, `read_inventory` | Questions catalogue, réponses stock |
| `read_checkouts` | Playbook `panier abandonné` |
| `read_returns` | Réponses politique de retour |
| `read_themes`, `write_themes` | Injection automatique du widget dans le thème actif via l'API `assets.json` |

<Note>
Les scopes `read_themes` / `write_themes` sont requis parce que sans eux, le widget de chat ne peut pas s'installer tout seul sur ta boutique. Actero écrit uniquement le snippet du widget — aucun autre fichier de thème n'est modifié.
</Note>

### Optionnels

| Scope | Feature débloquée si accordé |
|---|---|
| `write_orders` | Remboursements et annulations pilotés par l'agent |
| `read_draft_orders` | Contexte upsell dans les conversations |
| `read_shipping` | Réponses temps réel sur les tarifs de livraison |

Si tu décoches un scope optionnel sur l'écran Shopify, l'installation se poursuit et la feature correspondante reste désactivée tant que tu n'as pas réinstallé l'app pour la regranter.

<Warning>
Aucun scope de paiement n'est demandé. **Actero ne peut jamais encaisser, modifier un mode de paiement, ni accéder à tes données bancaires.** Les actions d'écriture (remboursement, annulation, devis) sont opt-in via les scopes optionnels et toujours auditables depuis Shopify Admin → **Apps**.
</Warning>

## Quelles données circulent

- **Commandes** (numéro, statut, total, items, tracking) → utilisées pour répondre aux WISMO et au SAV
- **Clients** (nom, email, langue, historique) → utilisés pour personnaliser le ton et reconnaître les clients VIP
- **Produits** (titre, description, variantes, prix, stock) → utilisés pour les questions catalogue
- **Paniers abandonnés** (items, total, email, URL de checkout) → utilisés pour les relances automatiques

Tout est stocké chiffré côté Actero (Supabase, RLS activée, isolation par `client_id`). Aucune donnée n'est jamais partagée avec un autre marchand.

## Webhooks configurés automatiquement

À l'installation, Actero enregistre cinq webhooks signés HMAC SHA-256 (vérification en temps constant via `crypto.timingSafeEqual`) :

| Webhook | Effet dans Actero |
|---|---|
| `orders/create` | Indexe la commande pour les futures réponses WISMO |
| `orders/updated` | Met à jour le statut tracking en temps réel |
| `carts/abandoned` | Déclenche le playbook `panier abandonné` |
| `customers/create` | Enrichit la base client (langue, géo, segment) |
| `app/uninstalled` | Coupe immédiatement la lecture Shopify si tu désinstalles l'app |

Tu peux désactiver un playbook depuis **Automatisation** sans toucher à l'intégration Shopify.

<Note>
Le webhook `app/uninstalled` est aussi déclaré côté Partner Dashboard pour respecter les exigences App Store. Le double enregistrement (Partner + API Admin) garantit que la désinstallation est toujours détectée, même en cas de mauvaise configuration côté Shopify.
</Note>

## Multi-boutique

Sur le plan **Entreprise**, une même organisation Actero peut connecter plusieurs boutiques Shopify et les gérer depuis un dashboard unifié.

Sur les plans **Free**, **Starter** et **Pro**, tu as **un seul domaine Shopify par compte Actero**. Si tu gères deux marques distinctes, crée un second compte Actero — chaque agent reste isolé et garde son ton, sa base de connaissances et ses métriques.

<Note>
Une seule plateforme e-commerce active par compte. Si tu connectes ensuite [WooCommerce](/integrations/woocommerce) ou [Webflow](/integrations/webflow), Actero te demande de déconnecter Shopify d'abord.
</Note>

## Déconnexion

Va dans **Intégrations → Shopify → Déconnecter**. Actero révoque immédiatement son token d'accès, supprime les quatre webhooks Shopify et arrête de lire tes données. Les conversations passées restent dans ton historique pour des raisons RGPD (export et suppression possibles depuis **Profil → Mes données**).

## Problèmes fréquents

<AccordionGroup>
  <Accordion title="L'agent ne trouve pas ma commande #1234">
    Vérifie que Shopify est bien en statut **Connecté** dans Intégrations. L'agent matche d'abord par email client, puis par numéro de commande. Si le client a passé sa commande en invité avec une autre adresse email que celle utilisée dans la conversation, demande-lui de confirmer son email d'achat.
  </Accordion>
  <Accordion title="Webhook error 401 ou 403 dans les logs Shopify">
    Cela arrive quand le secret HMAC a tourné (rotation de credentials côté Shopify). Déconnecte puis reconnecte l'intégration depuis Actero — les nouveaux webhooks sont régénérés avec un secret frais.
  </Accordion>
  <Accordion title="Relances panier abandonné qui ne partent pas">
    Trois choses à vérifier : (1) le playbook `panier abandonné` est **activé** dans Automatisation, (2) un service d'envoi est configuré ([Resend](/integrations/resend), [Gmail](/integrations/gmail) ou [SMTP/IMAP](/integrations/smtp-imap)), (3) le client a bien laissé son email à l'étape checkout (sans email, pas de relance possible).
  </Accordion>
  <Accordion title="J'ai changé de domaine Shopify (migration .myshopify.com)">
    Reconnecte l'intégration avec le nouveau domaine. Les commandes historiques restent indexées, le nouveau token prend le relais sans coupure de service.
  </Accordion>
  <Accordion title="Comment tester sans impacter mes vrais clients">
    Utilise **Tester mon agent** (Simulator) dans le dashboard. Les messages simulés n'envoient rien aux vrais clients, ne créent aucune commande et n'écrivent rien dans Shopify.
  </Accordion>
</AccordionGroup>

## FAQ

<AccordionGroup>
  <Accordion title="Actero apparaît-elle dans le Shopify App Store ?">
    Oui. L'installation passe par le flow OAuth officiel Shopify, ce qui te garantit que les scopes sont visibles, révocables à tout moment et auditables depuis Shopify Admin → **Apps**.
  </Accordion>
  <Accordion title="Combien de temps pour indexer toute ma boutique ?">
    Les 30 dernières commandes sont indexées en ~20 secondes (immédiat). L'indexation complète (catalogue, clients, historique 12 mois) prend de 2 à 30 minutes selon la taille de ta boutique. Tu reçois une notification dans le dashboard quand c'est fini.
  </Accordion>
  <Accordion title="Mes clients voient-ils que c'est une IA ?">
    Tu choisis. Par défaut, l'agent signe avec le nom de ta marque. Tu peux activer une signature explicite ("Réponse assistée par notre agent IA") depuis **Configurer l'agent → Identité**.
  </Accordion>
</AccordionGroup>
